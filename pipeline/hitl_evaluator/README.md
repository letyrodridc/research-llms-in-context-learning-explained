# Visual HITL Evaluation Dashboard

This directory contains the Human-in-the-Loop (HITL) evaluation tool designed to audit the logical and semantic quality of concept-based explanations generated by MLLMs (In-Context Learning).

The tool utilizes a hybrid architecture to maximize speed and facilitate collaborative work among researchers:
* **Frontend:** Hosted statically and for free on GitHub Pages.
* **Backend (Database):** Connected to Google Sheets via Google Apps Script (GAS) to centralize human evaluation scores in real-time.

## 🛠️ 1. Backend Deployment (Google Apps Script)

The backend is responsible for receiving evaluations from the interface, authenticating the researchers, and securely storing the data in a spreadsheet.

**Crucial Step:** The script MUST be created from inside a Google Sheet.
1. Open Google Sheets and create a new blank document (e.g., "HITL_ICML_Results").
2. From the top menu of the Google Sheet, go to **Extensions > Apps Script**. *(Do not go to script.google.com directly).*
3. Delete the default code and paste the content of the `Code.gs` file.
4. Update the `VALID_USERS` dictionary in the code with the actual usernames and passwords for your research team.
5. In the top menu, select the `setupSheet` function and click **Run**. This will automatically create the necessary headers in your spreadsheet. *(Note: Google will ask for authorization permissions the first time).*
6. Click on the blue button on the top right: **Deploy > New deployment**.
7. Configuration:
   * **Type:** Web app.
   * **Description:** HITL Backend v1.
   * **Execute as:** Me (your Google account).
   * **Who has access:** Anyone.
8. Click **Deploy**.
9. **⚠️ IMPORTANT:** Copy the generated **Web app URL**. You will need it for the Frontend configuration.

## 🚀 2. Frontend Deployment (GitHub Pages)

The frontend loads the *LLM-as-a-judge* results from the experiment's CSV files and connects with the HuggingFace Datasets API to render the visual context.

1. Open the `index.html` file in your code editor.
2. Go to approximately line `341`, and locate the `SCRIPT_URL` variable.
3. Replace the placeholder value with the Web app URL you copied from Google Apps Script in the previous step:
   ```javascript
   const SCRIPT_URL = "[https://script.google.com/macros/s/YOUR_ID_HERE/exec](https://script.google.com/macros/s/YOUR_ID_HERE/exec)";
```
4. Commit and push these changes to the `main` branch of the `research-llms-in-context-learning-explained` repository.
5. On GitHub, go to the **Settings > Pages** tab.
6. Under **Source**, select **Deploy from a branch**. Select the `main` branch and the `/root` folder (or if you configured GitHub Pages differently, make sure the path points to where your `index.html` is located).
7. Save. In a few minutes, GitHub will provide a public URL (e.g., `https://letyrodridc.github.io/research-llms-in-context-learning-explained/pipeline/hitl_evaluator/index.html`). This is the URL you should share with the human evaluators.

## 📊 3. Usage Instructions for Evaluators

1. Access the public GitHub Pages URL.
2. In the **Upload Evaluation Data** section, upload the `.csv` file generated by the evaluation pipeline (for example, the files located in `pipeline/openrouter_runs/.../judge_results.csv`).
3. The application will automatically process the metadata (Model, Dataset, Prompt) and extract the image index (`query_dataset_index`).
4. The tool will call the HuggingFace API (HF Datasets Server) to download and display the exact image corresponding to that trial.
   > **Fallback:** If the HuggingFace API fails or rate-limits requests, the evaluator can use the optional button to select the local folder containing the unzipped images.
5. The evaluator will read the explanation (DL Axioms, NLE, Rules, etc.), audit it using the 9 dimensions of the rubric, and click **Save and Next Trial**.
6. The data will instantly be sent to the configured Google Sheets.
