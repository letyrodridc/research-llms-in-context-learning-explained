# Visual HITL Evaluation Dashboard

This directory contains the Human-in-the-Loop (HITL) evaluation tool designed to audit the logical and semantic quality of concept-based explanations generated by MLLMs (In-Context Learning).

To maximize simplicity and eliminate CORS issues, this application is built as a **100% Google Apps Script Web App**. The frontend interface and the backend database are unified and hosted together on Google's infrastructure.

## 🛠️ 1. Deployment Instructions

The entire application runs from a single Google Sheets document.

**Crucial Step:** The script MUST be created from inside a Google Sheet.
1. Open Google Sheets and create a new blank document (e.g., "HITL_ICML_Results").
2. From the top menu of the Google Sheet, go to **Extensions > Apps Script**. *(Do not go to script.google.com directly).*
3. Create two files in the Apps Script editor:
   * `Code.gs` (paste the backend code here)
   * `Index.html` (click `+ > HTML` to create it, and paste the frontend code here).
4. Update the `VALID_USERS` dictionary inside `Code.gs` with the actual usernames and passwords for your research team.
5. In the top menu, select the `setupSheet` function and click **Run**. This will automatically create the necessary column headers in your spreadsheet. *(Note: Google will ask for authorization permissions the first time).*
6. Click on the blue button on the top right: **Deploy > New deployment**.
7. Configuration:
   * **Type:** Web app.
   * **Description:** HITL Evaluator v1.
   * **Execute as:** Me (your Google account).
   * **Who has access:** Anyone.
8. Click **Deploy**.
9. Google will provide you with a **Web app URL** (e.g., `https://script.google.com/macros/s/.../exec`). **This is the ONLY URL you need to share with your human evaluators.**

## 🗂️ 2. Optional: Local Fallback Setup (Recommended)

The dashboard automatically fetches images from the HuggingFace API. However, if the API rate-limits your requests or goes down, the dashboard can use a local folder of images as a fallback. 

To ensure the local images perfectly match the trials in the CSV, you must generate a `mapping.json` file using the provided Python script.

1. Locate the `generate_image_mapping.py` script.
2. Run the script from your terminal using the appropriate flags. You can process an entire directory or a single file.

   Process ALL files in a directory:
   
       python generate_image_mapping.py --dir /path/to/your/npy_folder/
   
   Process a SINGLE file:
   
       python generate_image_mapping.py --file /path/to/episode_N4_K5_Q1_run2.npy
   
   Specify a custom output path (optional):
   
       python generate_image_mapping.py --dir /path/to/npy_folder/ --output /path/to/save/mapping.json

3. Share the generated `mapping.json` file with your evaluators so they can upload it along with the local images folder if the HuggingFace API fails.

## 📊 3. Usage Instructions for Evaluators

1. Access the Web app URL provided by the lead researcher.
2. Log in using your assigned researcher credentials.
3. In the **Upload Evaluation Data** section, upload the `.csv` file generated by the evaluation pipeline (e.g., `judge_results.csv` or `trial_results.csv`).
4. **Local Fallback (If needed):** If images fail to load due to HF API limits, upload your local unzipped dataset folder and the `mapping.json` file in the fallback section.
5. The application will automatically process the metadata, extract the image index, and display the exact image corresponding to that trial.
6. Read the model's explanation (DL Axioms, NLE, Rules, etc.), audit it using the 9 dimensions of the rubric, and click **Save and Next Trial**.
7. The data will be instantly appended to the centralized Google Sheets document.

## 📈 4. Final Validation & Human-Machine Alignment (Data Analysis)

Once all evaluators have finished auditing the trials, the final step is to calculate the alignment between the human judgments and the autonomous LLM-as-a-judge. 

To account for natural human subjectivity in Likert scales, we use a **Relaxed Agreement Metric**:
* **1.0 points:** Exact match (Human Score == Judge Score).
* **0.75 points:** Partial agreement (Difference of exactly ± 1 point).
* **0.0 points:** Disagreement (Difference > 1).
* **Maximum Score per Explanation:** 9.0 (1.0 for each of the 9 evaluated dimensions).

### Execution Steps

1. **Export Human Data:** Go to your Google Sheets document, click on **File > Download > Comma Separated Values (.csv)**. Save this file as `HITL_ICML_Results.csv`.
2. **Prepare the Directory:** Ensure that your human results CSV, the judge's CSV (`judge_results.csv`), and the evaluation script (`calculate_alignment.py`) are in the same folder.
3. **Run the Analysis:** Execute the script from your terminal:

    python calculate_alignment.py

4. The script will output the average alignment per metric directly to your console (ready to be tabulated in your ICML paper) and will generate a new file called `HITL_Alignment_Results.csv` containing the row-by-row agreement breakdown.

## ⚠️ Troubleshooting

**Error:** `TypeError: Cannot read properties of null (reading 'getActiveSheet') setupSheet@ Code.gs:3`
* **Cause:** You created a Standalone Script instead of a Container-Bound Script. The code does not know which spreadsheet to write to.
* **Fix:** Close your current Apps Script tab. Open your actual Google Sheets document. Click on **Extensions > Apps Script** from the top menu of the spreadsheet. Paste your code there and click **Run** again.
